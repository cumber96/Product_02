## 문제 분석
`app.js`의 `renderSummaryCard()`가 표시하던 4개 항목(다음 예상 생리일 / 평균 주기 / 배란 예상일 / 가임기) 중
"다음 예상 생리일 / 배란 예상일 / 가임기"는 캘린더에 이미 색상·아이콘으로 마킹되어 있어 카드와 캘린더가 같은 정보를 중복 표시하고 있었음.
반면 사용자가 실제로 자주 확인하는 "평균 생리기간"과 "마지막 생리일"은 카드에 없었음.
`functions/_lib/prediction.js`의 `computePrediction()`은 이미 `avgPeriodLength`, `lastStartDate`를 계산해 API 응답에 포함하고 있었으므로, 백엔드 변경 없이 프론트엔드 표시만 바꾸면 되는 문제로 정의함.

## 수정 내용
- `app.js`: `renderSummaryCard()`의 `summary-grid` 항목을 "다음 예상 생리일 / 평균 주기 / 배란 예상일 / 가임기"(4개) → "평균 생리주기 / 평균 생리기간 / 마지막 생리일"(3개)로 교체. 이미 API가 내려주던 `avgCycleLength`, `avgPeriodLength`, `lastStartDate` 값을 그대로 사용.
- `style.css`: `.summary-grid`를 2열 그리드 → 3열 그리드(`repeat(3, 1fr)`)로 변경 (항목이 3개로 바뀌면서 마지막 줄에 빈 칸이 남는 것을 방지).

## 테스트 방법과 결과
1. `node --check app.js` : 구문 오류 없음.
2. `renderSummaryCard` 로직만 분리해 Node에서 mock 데이터(`hasData: true`/`false`)로 실행 → 3개 항목 HTML과 no-data 문구 모두 정상 출력 확인.
3. 로컬 D1(`wrangler d1 execute --local`)에 테스트 유저 1명과 생리 기록 4건(4~6월, 주기 28일·기간 5일 패턴)을 넣고 `wrangler pages dev`로 실행.
   - `curl /api/prediction` → `{"avgCycleLength":28,"avgPeriodLength":5,"lastStartDate":"2026-06-24", ...}` 확인.
4. Playwright(Chromium)로 세션 쿠키를 주입해 실제 홈 화면을 390px 모바일 뷰포트로 렌더링 → 예측 정보 카드에 "평균 생리주기 28일 / 평균 생리기간 5일 / 마지막 생리일 6월 24일"이 한 줄 3열로 정상 표시되고 레이아웃 깨짐 없음을 스크린샷으로 확인. 캘린더의 배란 예상일(💧)·가임기·예상 생리 마킹은 변경 없이 그대로 동작.
5. 테스트에 사용한 로컬 유저/세션/기록 데이터는 모두 삭제, `wrangler pages dev` 프로세스도 종료함.
