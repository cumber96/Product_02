## 문제 분석
기존 배란 예상일 마킹은 `.day-cell.ovulation::after`에 💧 이모지를 `font-size: 9px`로 날짜 칸 우측 상단에 절대 위치시키는 방식이었음.
- 다른 마킹(생리 기록/예상 생리/가임기)은 모두 원형 배경·테두리·음영 등 도형 기반의 일관된 스타일이었는데, 배란 예상일만 이모지를 사용해 시각적 톤이 어긋났음.
- 가임기(fertile) 배경이 있는 날짜와 겹칠 경우 이모지가 배경 위에서 더 두드러지지 않고 묻히는 문제도 있었음(실제 스크린샷에서 확인).
사용자에게 구체적 변경 방향을 확인한 결과, 기존 범례의 dot 스타일(`<i class="dot ...">`)과 통일된 방식으로 바꾸기로 결정.

## 수정 내용
- `app.js`
  - 범례의 `💧 배란 예상일` 텍스트를 `<i class="dot ovulation"></i>배란 예상일`로 교체 (다른 범례 항목과 동일한 마크업 패턴).
- `style.css`
  - `.dot.ovulation { background: var(--luxe); }` 추가 — 가임기 배경색(`--luxe-soft`)과 같은 색 계열의 진한 버전을 사용해, "가임기(연한 배경) 안의 배란일(진한 점)"이라는 관계가 시각적으로 드러나도록 함.
  - `.day-cell.ovulation::after`를 이모지 대신 5px 크기의 채워진 원(`--luxe` 색상)으로 변경, 날짜 칸 하단 중앙에 배치해 날짜 숫자와 겹치지 않게 함.
